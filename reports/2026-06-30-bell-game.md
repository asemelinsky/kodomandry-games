# Гра 5 — «Не дзвони у дзвіночок» (bell)

**Дата:** 2026-06-30
**Сесія:** vps-dev
**Статус:** код готовий + unit-tested, чекає browser/phone-тест і approve на deploy

## Що це

П'ята гра у `kodomandry-games` pack. Перша **реактивна (real-time)** гра — решта (RPS / TTT / Memory / Battleship) turn-based. Натхнення — Roblox-серія ігор на двох за столом (Олексій описав 23.06).

## Механіка (узгоджено з Олексієм)

- Дзвіночок схований під стаканчиком. У кожного гравця **3 ❤**.
- Після старту автоматична анімація: руки по черзі «торкаються» стаканчика (**beat**), темп зростає (інтервал між beat скорочується: 1500 → 820 мс, крок −70).
- Гравець керує лише своїми двома кнопками:
  - 💚 **Підняти** — на своєму **наступному** beat рука підіймає стаканчик → дзвіночок видно опоненту.
  - 🛑 **Стоп** — на своєму наступному beat рука **не** торкається (захист).
- Підняв X → небезпечний beat = наступний (опонента Y). Якщо Y не натиснув 🛑 на цей beat → 🔔 **дзвінок** → Y втрачає ❤. Встиг 🛑 → захист, темп **reset до старту**, гра триває.
- Хто перший втратив усі 3 ❤ — **програв**.

### Узгоджені деталі
1. **Cooldown 3 с PER-BUTTON (варіант A)** — натиснув 💚 → замерзає лише 💚, 🛑 ще працює. Більше тактичної глибини (можна 💚 + одразу 🛑). *(Олексій обрав A 30.06.)*
2. 🛑 одночасно обома — **не конфлікт**, кожен керує своєю рукою.
3. Стаканчик — ресурс на двох: 💚 **first-click-wins** за timestamp; другий натиснув впусту (але його кнопка все одно йде в cooldown).
4. Після дзвінка — пауза ~1.8 с з «🔔 ДЗВІНОК! −1❤», тоді автозапуск нового прогону з reset темпу (без зайвого кліку). Стартер чергується.

## Архітектура

Той самий патерн, що й решта pack:
- **`bell/index.html`** — клієнт (vanilla JS). Анімація — `requestAnimationFrame`, обчислюється локально від `roundStartAt` + `serverOffset` (server clock alignment). Polling `GET` кожні **350 мс** (швидше за turn-based ігри — потрібно для реакції). Per-button cooldown показано circular-fill на кнопці.
- **`api/bell.js`** — Vercel serverless, стан в Upstash Redis (`bell:room:<id>`, TTL 24 год). **Server-authoritative** з **лінивою резолюцією**: beat рахується детерміновано від `roundStartAt`; підняття/дзвінок/захист вирішує сервер на кожному GET/POST за серверним `Date.now()`. Чесність судить сервер, не клієнт.
- **`vercel.json`** — rewrite `/bell` → `/bell/index.html`.
- **`index.html`** (landing) — додано 5-ту картку 🔔.

### Чому server-authoritative + deterministic clock
Реакційна гра з 300–1500 мс вікнами не може покладатись лише на polling (надто грубо) чи лише на клієнт (десинк/чіт). Рішення: клієнт малює анімацію локально (плавно), але всі ігрові події (підняв/стоп/дзвінок) фіксує/судить сервер за timestamp-ами. `MIN beat = 820 мс` навмисно тримає вікно реакції ≥ poll+latency, щоб гра лишалась грабельною для дітей.
